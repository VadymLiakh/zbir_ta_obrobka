У даній практичній роботі було реалізовано обробку великих CSV-файлів розміром понад 1GB без перевантаження оперативної пам’яті.

Робота складається з трьох основних частин. Спочатку за допомогою скрипта generate_big_csv.py було згенеровано два великі файли (big_left.csv і big_right.csv). Дані створюються потоково, рядок за рядком, що дозволяє уникнути великого навантаження на пам’ять.

Далі у скрипті merge_big_csv.py виконано об’єднання файлів по колонці ID. Для цього використано підхід із chunksize, який дозволяє читати великий файл частинами. Об’єднання виконано типом LEFT JOIN, тобто всі рядки з лівого файлу зберігаються. Результат записується у файл merged_result.csv також поступово.

У скрипті verify_merge.py виконано перевірку результату. Перевіряється кількість рядків, наявність колонок із правого файлу, а також коректність об’єднання на вибіркових значеннях ID.

У результаті було підтверджено, що обробка великих даних може виконуватись ефективно за допомогою потокового підходу без перевантаження системи, а об’єднання файлів виконано коректно.
